Html Cheatsheet

1.- The a tag

It defines a placeholder for a hyperlink, this changes depending on the attribute. The most common use of it is with the "href" attribute.

The different uses of the <a> tag are:

Attribute Value Description Code
download filename Makes a hyperlink that downloads a file when clicked. 
<a> href="file" download </a>
href URL Specifies the URL the hyperlink goes to. 
<a href="link"> </a>

<a href="mailto:someone@example.com"> mailto link </a>

<a href="tel:+4733378901"> +47 333 78 901 </a>

<a href="#id"> text </a>

<a href="javascript:alert"> execute JavaScript </a>

Email me

2.- The p tag

This tag defines a paragraph and creates a blank line before and after each <p> tag.

This is a paragraph.

<p> paragraph text </p>

3.- The h1-h6 tags

This tag defines html headings, <h1> being the most important an <h6> being the least important.

<h1>This is a heading 1</h1>

This is a heading 1

 
<h2>This is a heading 2</h2>

This is a heading 2

 
<h3>This is a heading 3</h3>

This is a heading 3

 
<h4>This is a heading 4</h4>

This is a heading 4

 
<h5>This is a heading 5</h5>
This is a heading 5
 
<h6>This is a heading 6</h6>
This is a heading 6

4.- The div tag

This tag creates a division or section in an html document, it contains html elements that are then styled with "CSS" with class or id atribute or manipulated with "JavaScript".

This is an html div

I am CSS and JavaScript compatible!

<div class="Div1">
  <h2>Div heading</h2>
  <p>Div text</p>
</div>

5.- The span tag

This tag is an inline container that marks a part of a text or document that is easilystyled with "CSS" or manipulated with "JavaScript" using a "class" or "id" attributes.

This text has more than one color. 
<p><span style="color:blue">This text has</span>more<span style="color:green">than one color</span>

6.- The strong tag

This tag defines text of greater importance, it is typically displayed in bold, not to be confused with the <b> tag which only makes the text bold without extra importance.

This is important text! 
<strong>This is important text!</strong>

7.- The i tag

This tag defines text in an alternate voice or mood, typically displayed in italics, it is often usedd for technical terms, phrases in another languages, thoughts, names, among others.

This text is in italics. 
<i>This text is in italics</i>

8.- The u tag

This tag defines underlined, unarticulated or differently styled text, can be used for misspelled words or proper names in Chinese text, its default is to be displayed as underlined text but this can be changed with "CSS".

This text is underlined. 
<p>This text is <u>mispeled</u> text</p>

9.- The b tag

This tag specifies bold text without extra importance, it is best to avoid using it according to "HTML5 documentation" since there are many other ways to specify text with different importance.

This is bold text 
<b>This is bold text</b>

10.- The strike tag

This tag defined strikethrough text in "HTML4" but was discontinued for "HTML5" and replaced with the <del> tag. A common use for it is in sold out items for example.

Text that is no longer valid. 
<strike>Text that is no longer valid</strike>

10.1.- The del tag

This tag defines text that has been deleted from a document, typically browsers strike a line through said text.

My favorite color is green.

<p>My favorite color is <del>blue</del> red </p>

10.2.- The ins tag

This tag normally goes hand in hand with the del tag, since it defines new inserted text in a document, browsers usually underline said text.

My favorite color is green blue.

<p>My favorite color is <del>blue</del> <ins>red</ins> </p>

11.- The small tag

This tag defines smaller text that could be used for copyright or other side-comments, it is not a forbiden tag but you could achieve the same or an even better effect by using "CSS".

Normal text

This is small text 
<p>This is normal text.</p>
<p><small>This is smaller text.</small></p>

12.- The em tag

This tag is used to define emphasized text, its content is usually displayed in italics.

You have to hurry up!

<p>You <em>have</em> to hurry up!</p>

13.- The header tag

This tag represents a container for introductory content or a set of navigational links. It typically contains:

  • One or more heading elements (<h1> - <h6>)
  • Logo or icon
  • Authorship information
Although there can be many different <header> elements in one html document, they can't be used within a <footer>, <address> or another <header> element.

A heading here

Posted by Me

Additional information

<header>
  <h1>A heading here</h1>
  <p>Posted by Me</p>
  <p>Additional information</p>
</header>

14.- The section tag

This tag defines a section in a document.

Company History

Description

<section>
  <h2>Company History</h2>
  <p>Description</p>
</section>

15.- The article tag

This tag specifies independent, self-contained content. It is text that should make sense on its own and should be distributed independently from the site, some sources for the <article> element include:

  • Forum post
  • Blog post
  • News story
This element does not render anything on its own but it can be styled with "CSS".

Google Chrome

Google Chrome is a web broser developped by Google

<article>
  <h2>Google Chrome</h2>
  <p>Google Chrome is a web broser developped by Google</p>
</article>

16.- The aside tag

This tag defines content aside from the content it is in, it should be indirectly related to surrounding content, it is usually some sort of sidebar in a document. This tag also doesn't render anything on itd own but can be styled using "CSS".

This one time I visited the Epcot center

Epcot Center

Epcot is a theme park at Walt Disney Resort

<p>This one time I visited the Epcot center</p>

<aside>
  <h4>Epcot Center</h4>
  <p>Epcot is a theme park at Walt Disney Resort</p>
</aside>

17.- The footer tag

This tag defines a footer for a document or section. They typically contain:

  • Authorship information
  • Copyright information
  • Constact information
  • Sitemap
  • Back to top links
  • Related documents
You can have several footnote elements in a single document.

Author: Name

name@example.com

<footer>
  <p>Author: Name</p>
  <p><a href="mailto:name@example.com">name@example.com</a></p>
</footer>

18.- The nav tag

This tag defines a set of navigation links, it is an element intended only for major blocks of navigation links and some browsers can use this element to choos to omit initial rendering of this content.

HTML CSS JavaScript>/a> Python 
<nav>
  <a href="/html/">HTML</a>
  <a href="/css/">CSS</a>
  <a href="/js/">JavaScript</a>
  <a href="/python/">Python</a>
</nav>

19.- The img tag

This tag embeds an image in an html page, they are technically not inserted since they are linked to web pages. This element is simply a holding space for the referenced image. It requires the following attributes:

  • src - specifies the path to the image.
  • alt - specifies alternate text for the image if it can[t be displayed.
  • height - specifies dimensions of the image.
  • width - specifies dimensions of the image.
This tag can be nested inside the <a> tag to use the image as a link.

<img src="img.png" alt="img description" width="900" height="900">

20.- The br tag

This tag inserts a single line break and since it is an empty tag it has no end tag.

This
Text
Has
Line
Breaks

<br>

21.- The hr tag

This tag creates a horizontal rule that separates content, marking a thematic break in an html page. As is the case with the <br> tag, it has no end tag.

<hr>

22.- The form tag

This tag creates an html form for the user to fill, it can contain one or more of these elements: 

  • <input>
  • <textarea>
  • <button>
  • <select>
  • <option>
  • <optgroup>
  • <fieldset>
  • <label>
  • <output>





<form action="/action_page.php" method="get">
    <label for="fname">First name:</label>
    <input type="text" id="fname" name="fname"><br><br>
    <label for="lname">Last name:</label>
    <input type="text" id="lname" name="lname"><br><br>
    <input type="submit" value="Submit">
</form>

22.1.- The input tag

This tag specifies an input field where the user can enter data, this is the core element used to create <form> elements. There are many different input types, and they are as follows:

Type Description Code
<button> defines a clickable button (used with a "JavaScript" to activate it). 
<input type="button" value="Click me" onclick="msg()">

<script>
function msg() {
  alert("Hello world!");
}
<checkbox> defines a checkbox that is shown as a box that when activated is ticked, they allow users to select one or more options among an arbitrary number of choices. 
<input type="checkbox" id="vehicle1" name="vehicle1" value="Bike">
<color> defines a color picker in seven character hexadecimal notation. 
<input type="color" id="favcolor" name="favcolor" value="#ff0000">
<date> defines a date picker with the values: day, month and year. 
<input type="date" id="birthday" name="birthday">
<datetime-local> defines a date picker with the values: day, month, year and time. 
<input type="datetime-local" id="birthdaytime" name="birthdaytime">
<email> defines a field for an email address that is automatically validated to ensure proper format. For multiple email addresses, add the "multiple". 
  • <input type="email" id="email" name="email">
  • <input type="email" id="emails" name="emails" multiple>
<file> defines a file-select field and a "Browse" for the file uploads, can add multiple. 
  • <input type="file" id="myfile" name="myfile">
  • <input type="file" id="myfile" name="myfile" multiple>
<submit> defines a submit button that submits all form values to a form handler. 
<input type="submit">
<image> defines an image as a submit button with "src" being the path to the image. 
<input type="image" src="img_submit.gif" alt="Submit" width="48" height="48">
<number> defines a field for entering a number. To specify restrictions:
  • max - specifies the maximum value allowed.
  • min - specifies the minimum value allowed.
  • step - specifies the number intervals
  • value - specifies default value
 
<input type="number" id="quantity" name="quantity" min="1" max="5" step="1">
<password> defines a password field where characters are masked 
<input type="password" id="pwd" name="pwd">
<radio> defines a radio button, they are normally presented in groups where only one can be selected at the same time. 
<input type="radio" id="html" name="fav_language" value="HTML">
<input type="radio" id="html" name="fav_language" value="CSS">
<input type="radio" id="html" name="fav_language" value="JavaScript">
<range> defines a slider that gives a number that doesn't have to be exact. Its default range is from 0 to 100. To specify restrictions:
  • max - specifies the maximum value allowed.
  • min - specifies the minimum value allowed.
  • step - specifies the number intervals
  • value - specifies default value
 
<input type="range" id="vol" name="vol" min="0" max="50">
<reset> defines a reset button to reset form values to their initial ones. 
<input type="reset">
<tel> defines a field for entering a telephone number, if not supported, use "text". 
<input type="tel" id="phone" name="phone" pattern="[0-9]{3}-[0-9]{2}-[0-9]{3}">
<text> defines a single-line text field, its default width is 20 characters. 
<input type="text" id="fname" name="fname">

22.2.- The textarea tag

               
This tag defines a multi-line text input 
                control, it is often used in a 
                form for comments or reviews. This tag can 
                hold an unlimited amount of characters and 
                its text renders in a fixed width font 
                (usually Courier). The size of a text area is 
                modified by the "cols" and 
                "rows" attributes or 
                through "CSS". The 
                "name" attribute is required 
                for referencing the form data after submission, 
                if ommited, no data will be submitted.

               
<textarea id="w3review" name="w3review" rows="4" cols="50">

This tag defines a clickable button, 
                  inside this element, text and tags like 
                  <i>, <b>, 
                  <strong>, <br>, 
                  <img>. It is a good practice 
                  to specify the type in order to let the 
                  browser know what type of button it is.
                  There are three different types of buttons:
                  
    
                    
  • "submit" - submits the current form data (default).
    • 
                    
  • "reset" - resets data in current form.
    • 
                    
  • "button" - a basic button that is controlled with "JavaScript".
    • 
                  

                  
    
                    
  • <button type="button" onclick="alert('This button does nothing.')">Click Me</button>
    • 
                    
  • <button type="reset">Reset the form!</button>

22.4.- The select tag

               
This tag creates a drop down list, it is 
                  often used in a form to collect user input, 
                  as is the case with the <textarea> 
                  tag, The "name" attribute is 
                  required for referencing the form data after 
                  submission, if ommited, no data will be 
                  submitted. It is built through a series of 
                  <option> tags.

               
<select id="cars">
  <option value="volvo">Volvo</option>
  <option value="saab">Saab</option>
  <option value="opel">Opel</option>
  <option value="audi">Audi</option>
</select>


               
22.5.- The option tag

               
This tag defines an option in a select list, 
                these go inside <select>, 
                <optgroup>, or <datalist> 
                elements, this tag can be used without 
                attributes but value is usually needed for 
                data to be sent to the server.

               
<option value="volvo">Volvo</option>


               
22.6.- The optgroup tag

               
This tag is used to group related options in 
                a <select> element, its use 
                is prudent when the list of options is long.

               
<optgroup label="Swedish Cars">
  <option value="volvo">Volvo</option>
  <option value="saab">Saab</option>
</optgroup>


               
22.7.- The fieldset tag

               
This tag groups related elements in a form, 
                take all the information of employees for 
                example. The <legend> tag is 
                used to define the caption of this element.

               
<fieldset>
  <legend>Employee Information:</legend>
</fieldset>

It defines a label for most input types, 
                  for the <meter>, 
                  <progress>, <select> 
                  and <textarea> tags the use of 
                  labels has two main benefits:
                  
    
                    
  • The label is read out loud when the user is focused on the element.
    • 
                    
  • It increases hit area of the element.
    • 
                  

               

               
<label for="id">id</label>
